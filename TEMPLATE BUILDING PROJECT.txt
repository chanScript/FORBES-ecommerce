> **Stack:** React (Vite) + TailwindCSS | Node.js + Express | PRISMA ORM | POSTGRES | JWT Auth | RBAC | Nodemailer
>
> **Hosting Target:** cPanel (Node.js Application)

---

## 1. System Overview

This system is a **secure, scalable, role-based web application** designed with a clear separation between **Client (Frontend)** and **Server (Backend)**. It supports:

* JWT-based authentication
* Role & permission-based authorization (**Super Admin / Admin / User**)
* **Trashbin (Soft Delete) system for sub-domains**
* Audit trail logging for compliance and traceability
* Optimized database querying with pagination & filtering
* Email notifications via Nodemailer
* Production-ready structure suitable for **cPanel deployment**

## 7. Trashbin (Soft Delete) System – Sub Domains

### Purpose

Prevents permanent deletion while allowing **Admin** and **Super Admin** to manage recovery and cleanup safely.

---

### Deletion Rules

* **Users**: ❌ No delete access
* **Admin**: ✅ Can soft-delete & restore
* **Super Admin**: ✅ Full control (delete, restore, permanent purge)

---

### Soft Delete Logic

* Record is **never removed** from database
* `isDeleted = 1` marks record as trashed
* `deletedAt` & `deletedBy` are populated

---

### Default Query Rule (IMPORTANT)

All standard queries **MUST include**:

```
WHERE isDeleted = 0
```

This ensures trashed records are **never visible** to normal users.

---

### Trashbin Endpoints (Admin / Super Admin Only)

```
GET    /api/sub-domains/trash        # List trashed records
PATCH  /api/sub-domains/:id/restore  # Restore record
DELETE /api/sub-domains/:id/force    # Permanent delete (Super Admin only)
```

---

## 8. Trash Middleware

### trash.middleware.js

Responsibilities:

* Auto-filter `isDeleted = 0`
* Block access to deleted records for non-admins
* Enforce role-based restore & purge

Middleware Order:

```
JWT → RBAC → Trash Check → Controller
```
